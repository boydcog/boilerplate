# 바이브코딩 가이드 📋✨

> **바이브코딩이란?** 기획안과 디자인을 바탕으로 AI와 함께 빠르게 웹 애플리케이션을 개발하는 방법입니다.

## 🎯 이 가이드의 목적

- **기획자**: 개발을 몰라도 아이디어를 실제 웹 서비스로 만들 수 있습니다
- **개발자**: 기획안과 디자인을 효율적으로 코드로 변환할 수 있습니다
- **팀**: 기획부터 개발까지 원활한 협업이 가능합니다

## 📝 기획안 준비하기

### 1. 기획안 작성 템플릿

`docs/prd/` 폴더에 다음과 같은 구조로 기획안을 작성하세요:

```markdown
# [프로젝트명] 기획안

## 📋 프로젝트 개요
- **목적**: 무엇을 위한 서비스인가요?
- **타겟 사용자**: 누가 사용하나요?
- **핵심 가치**: 어떤 문제를 해결하나요?

## 🎯 주요 기능
1. **기능 1**: 사용자 로그인
   - 이메일/비밀번호로 로그인
   - 소셜 로그인 (구글, 카카오)
   
2. **기능 2**: 게시글 작성
   - 제목, 내용, 이미지 업로드
   - 카테고리 선택

## 👥 유저스토리
- **사용자로서** 로그인을 하고 싶다 **왜냐하면** 내 정보를 안전하게 관리하고 싶기 때문이다
- **사용자로서** 게시글을 작성하고 싶다 **왜냐하면** 다른 사람들과 정보를 공유하고 싶기 때문이다

## 📊 성공 지표
- 월간 활성 사용자 1,000명
- 게시글 작성률 70%
```

### 2. 유저스토리 세분화

`docs/userstory/` 폴더에 각 기능별로 상세한 유저스토리를 작성하세요:

**예시: `docs/userstory/로그인.md`**
```markdown
# 로그인 기능 유저스토리

## 📱 화면별 스토리

### 로그인 화면
- **사용자로서** 이메일과 비밀번호를 입력하고 싶다
- **사용자로서** "로그인 유지" 옵션을 선택하고 싶다
- **사용자로서** 비밀번호를 잊었을 때 재설정하고 싶다

### 회원가입 화면
- **사용자로서** 간단한 정보로 회원가입하고 싶다
- **사용자로서** 이메일 인증을 받고 싶다

## ✅ 수용 기준 (Acceptance Criteria)
- [ ] 올바른 이메일/비밀번호로 로그인 성공
- [ ] 잘못된 정보 입력 시 에러 메시지 표시
- [ ] 로그인 후 메인 페이지로 이동
- [ ] 로그인 상태 유지 기능
```

## 🎨 Figma 디자인 연동하기

### 1. Figma API 토큰 발급

1. **Figma 접속** → 우상단 프로필 클릭
2. **Settings** → **Personal access tokens**
3. **Generate new token** 클릭
4. 토큰 이름 입력 후 생성
5. 생성된 토큰 복사 (⚠️ 한 번만 보여집니다!)

### 2. 프로젝트에 Figma 연동 설정

**개발자가 할 일:**

1. `.env` 파일에 Figma 정보 추가:
```bash
# Figma 연동
FIGMA_ACCESS_TOKEN=your_figma_token_here
FIGMA_FILE_KEY=your_figma_file_key_here
```

2. Figma 파일 키 찾기:
   - Figma 파일 URL: `https://www.figma.com/file/ABC123/프로젝트명`
   - 파일 키: `ABC123` 부분

### 3. 디자인 시스템 구성 가이드

**기획자/디자이너가 Figma에서 준비할 것:**

```
📁 Figma 파일 구조
├── 🎨 Design System
│   ├── Colors (색상 팔레트)
│   ├── Typography (폰트 스타일)
│   ├── Components (버튼, 입력창 등)
│   └── Icons (아이콘 세트)
├── 📱 Screens
│   ├── 로그인 화면
│   ├── 메인 화면
│   ├── 게시글 목록
│   └── 게시글 작성
└── 🔄 User Flow
    └── 사용자 여정 다이어그램
```

**중요한 네이밍 규칙:**
- 화면명: `Screen/로그인`, `Screen/메인`
- 컴포넌트: `Button/Primary`, `Input/Text`
- 색상: `Primary/Blue`, `Secondary/Gray`

## 🤖 AI와 함께 개발하기

### 1. 개발 시작하기

**개발자가 실행할 명령어:**
```bash
# 프로젝트 초기 설정
make setup

# 개발 서버 시작
make dev-integrated
```

### 2. AI에게 작업 요청하는 방법

**효과적인 프롬프트 예시:**

```
📋 기획안을 바탕으로 로그인 기능을 개발해주세요.

📁 참고 자료:
- 기획안: docs/prd/프로젝트명.md
- 유저스토리: docs/userstory/로그인.md
- Figma 디자인: [Figma 링크]

🎯 요구사항:
1. 이메일/비밀번호 로그인 폼
2. 입력 검증 및 에러 처리
3. 로그인 성공 시 메인 페이지 이동
4. Figma 디자인과 동일한 UI

🔧 기술 스택:
- Frontend: React + TypeScript
- Backend: FastAPI + PostgreSQL
- 스타일링: Tailwind CSS
```

### 3. 단계별 개발 프로세스

**1단계: 데이터 모델 설계**
```
사용자 테이블 설계:
- id (고유번호)
- email (이메일)
- password (암호화된 비밀번호)
- created_at (가입일시)
```

**2단계: API 엔드포인트 개발**
```
POST /api/auth/login    - 로그인
POST /api/auth/register - 회원가입
GET  /api/auth/me       - 내 정보 조회
POST /api/auth/logout   - 로그아웃
```

**3단계: 프론트엔드 화면 개발**
```
src/pages/LoginPage.tsx     - 로그인 페이지
src/components/LoginForm.tsx - 로그인 폼 컴포넌트
src/hooks/useAuth.ts        - 인증 관련 훅
```

**4단계: 통합 테스트**
```bash
# 테스트 실행
make test

# 코드 품질 검사
make lint
make format
```

## 🔍 결과물 확인하기

### 1. 개발 서버에서 확인

개발 중인 웹사이트 확인:
- **웹사이트**: http://localhost:8000
- **API 문서**: http://localhost:8000/docs

### 2. 기능 체크리스트

**기획자가 확인할 항목:**
- [ ] 기획안의 모든 기능이 구현되었는가?
- [ ] 유저스토리대로 동작하는가?
- [ ] Figma 디자인과 일치하는가?
- [ ] 사용자 경험이 자연스러운가?

**개발자가 확인할 항목:**
- [ ] 모든 테스트가 통과하는가?
- [ ] 코드 품질 기준을 만족하는가?
- [ ] 성능 이슈는 없는가?
- [ ] 보안 취약점은 없는가?

### 3. 피드백 및 수정

**피드백 방법:**
1. **GitHub Issues**: 버그나 개선사항 등록
2. **ClickUp**: 작업 진행상황 관리
3. **Figma Comments**: 디자인 관련 피드백

## 🤝 협업 팁

### 기획자를 위한 팁

**✅ 좋은 기획안 작성법:**
- 구체적이고 명확한 요구사항 작성
- 사용자 관점에서 기능 설명
- 예외 상황과 에러 케이스 고려
- 우선순위 명시 (Must have, Should have, Could have)

**✅ 효과적인 소통 방법:**
- 개발 용어보다는 사용자 관점의 언어 사용
- 스크린샷이나 화면 녹화로 설명
- "왜 이 기능이 필요한지" 배경 설명

### 개발자를 위한 팁

**✅ 기획안 이해하기:**
- 기획 의도와 비즈니스 목표 파악
- 불명확한 부분은 적극적으로 질문
- 기술적 제약사항이 있다면 대안 제시

**✅ AI 활용 극대화:**
- 기획안과 디자인을 AI에게 충분히 설명
- 단계별로 나누어 개발 요청
- 코드 리뷰와 테스트를 AI에게 요청

### 공통 협업 가이드

**📅 개발 일정 관리:**
```
1주차: 기획안 검토 및 기술 설계
2주차: 핵심 기능 개발 (MVP)
3주차: UI/UX 완성 및 테스트
4주차: 배포 및 피드백 반영
```

**🔄 정기 미팅:**
- **Daily Standup** (15분): 진행상황 공유
- **Weekly Review** (1시간): 완성된 기능 리뷰
- **Sprint Planning** (2시간): 다음 주 계획 수립
