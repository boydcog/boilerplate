# 기획안 (PRD - Product Requirements Document)

이 폴더에는 프로젝트의 기획안을 저장합니다.

## 📁 폴더 구조

```
docs/prd/
├── README.md           ← 이 파일
├── 프로젝트명.md        ← 메인 기획안
├── 기능명세서.md        ← 상세 기능 명세
└── 데이터모델.md        ← 데이터 구조 설계
```

## 📝 기획안 작성 가이드

### 1. 메인 기획안 (`프로젝트명.md`)
- 프로젝트 개요 및 목적
- 타겟 사용자 정의
- 핵심 기능 목록
- 비즈니스 목표 및 성공 지표

### 2. 기능 명세서 (`기능명세서.md`)
- 각 기능별 상세 요구사항
- 사용자 인터페이스 설명
- 비즈니스 로직 정의
- 예외 상황 처리 방안

### 3. 데이터 모델 (`데이터모델.md`)
- 데이터베이스 테이블 구조
- 데이터 관계 정의
- API 엔드포인트 명세
- 데이터 검증 규칙

## 🎯 작성 팁

- **구체적으로**: 모호한 표현보다는 명확한 요구사항
- **사용자 중심**: 개발자 관점이 아닌 사용자 관점에서 작성
- **우선순위**: Must have, Should have, Could have로 구분
- **예시 포함**: 실제 사용 시나리오나 예시 데이터 포함

## 📋 템플릿

새로운 기획안을 작성할 때는 `docs/바이브코딩_가이드.md`의 템플릿을 참고하세요.
