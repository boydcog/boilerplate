# 블로그 플랫폼 유저스토리

## 🔐 사용자 인증

### 회원가입
- **방문자로서** 이메일과 비밀번호로 회원가입하고 싶다 **왜냐하면** 내 블로그를 시작하고 싶기 때문이다
- **방문자로서** 구글 계정으로 간편 가입하고 싶다 **왜냐하면** 복잡한 가입 절차를 피하고 싶기 때문이다
- **신규 사용자로서** 가입 후 바로 첫 글을 쓸 수 있기를 원한다 **왜냐하면** 빠르게 블로그를 시작하고 싶기 때문이다

#### 수용 기준
- [ ] 이메일 중복 검사 및 유효성 검증
- [ ] 비밀번호 강도 검사 (8자 이상, 영문+숫자)
- [ ] 구글 OAuth 로그인 연동
- [ ] 가입 완료 후 프로필 설정 페이지로 이동
- [ ] 이메일 인증 (선택사항)

### 로그인
- **사용자로서** 이메일과 비밀번호로 로그인하고 싶다 **왜냐하면** 내 계정에 접근하고 싶기 때문이다
- **사용자로서** 로그인 상태를 유지하고 싶다 **왜냐하면** 매번 로그인하는 것이 번거롭기 때문이다
- **사용자로서** 비밀번호를 잊었을 때 재설정하고 싶다 **왜냐하면** 계정에 다시 접근하고 싶기 때문이다

#### 수용 기준
- [ ] 올바른 이메일/비밀번호로 로그인 성공
- [ ] 잘못된 정보 입력 시 명확한 에러 메시지
- [ ] "로그인 유지" 체크박스 (7일간 유지)
- [ ] 비밀번호 찾기 이메일 발송
- [ ] 로그인 후 이전 페이지로 이동

## ✍️ 글 작성 및 관리

### 글 작성
- **블로거로서** 마크다운으로 글을 작성하고 싶다 **왜냐하면** 코드와 텍스트를 아름답게 표현하고 싶기 때문이다
- **블로거로서** 실시간으로 미리보기를 보고 싶다 **왜냐하면** 최종 결과물을 확인하며 작성하고 싶기 때문이다
- **블로거로서** 이미지를 쉽게 업로드하고 싶다 **왜냐하면** 시각적인 콘텐츠를 포함하고 싶기 때문이다
- **블로거로서** 글을 임시저장하고 싶다 **왜냐하면** 나중에 이어서 작성하고 싶기 때문이다

#### 수용 기준
- [ ] 마크다운 문법 지원 (제목, 목록, 코드블록, 링크 등)
- [ ] 실시간 미리보기 패널
- [ ] 드래그앤드롭 이미지 업로드
- [ ] 코드 하이라이팅 (다양한 언어 지원)
- [ ] 자동 임시저장 (30초마다)
- [ ] 글 제목, 태그, 카테고리 설정
- [ ] 발행/비공개 상태 선택

### 글 목록 및 관리
- **블로거로서** 내가 작성한 글 목록을 보고 싶다 **왜냐하면** 내 콘텐츠를 관리하고 싶기 때문이다
- **블로거로서** 글을 수정하고 삭제하고 싶다 **왜냐하면** 콘텐츠를 업데이트하고 관리하고 싶기 때문이다
- **블로거로서** 글의 조회수와 좋아요 수를 보고 싶다 **왜냐하면** 독자들의 반응을 알고 싶기 때문이다

#### 수용 기준
- [ ] 내 글 목록 페이지 (제목, 작성일, 상태, 조회수)
- [ ] 글 상태별 필터링 (발행됨, 임시저장, 비공개)
- [ ] 글 검색 기능 (제목, 내용)
- [ ] 글 수정 페이지 (기존 내용 불러오기)
- [ ] 글 삭제 확인 다이얼로그
- [ ] 조회수, 좋아요 수, 댓글 수 표시

## 📖 글 읽기 및 탐색

### 글 상세 보기
- **독자로서** 글을 편안하게 읽고 싶다 **왜냐하면** 좋은 콘텐츠를 즐기고 싶기 때문이다
- **독자로서** 코드가 예쁘게 표시되기를 원한다 **왜냐하면** 기술 블로그의 코드를 이해하고 싶기 때문이다
- **독자로서** 글 작성자 정보를 보고 싶다 **왜냐하면** 작성자에 대해 더 알고 싶기 때문이다

#### 수용 기준
- [ ] 깔끔한 타이포그래피 (제목, 본문, 코드)
- [ ] 코드 블록 하이라이팅 및 복사 버튼
- [ ] 작성자 프로필 카드 (이름, 사진, 소개)
- [ ] 작성일, 수정일, 읽는 시간 표시
- [ ] 태그 목록 표시
- [ ] 관련 글 추천 (하단)

### 글 목록 탐색
- **방문자로서** 최신 글들을 보고 싶다 **왜냐하면** 새로운 콘텐츠를 발견하고 싶기 때문이다
- **방문자로서** 인기 글들을 보고 싶다 **왜냐하면** 많은 사람들이 좋아하는 콘텐츠를 보고 싶기 때문이다
- **방문자로서** 특정 주제의 글을 찾고 싶다 **왜냐하면** 관심 있는 분야의 글을 읽고 싶기 때문이다

#### 수용 기준
- [ ] 메인 페이지에 최신 글 목록 (10개)
- [ ] 인기 글 섹션 (좋아요 수 기준)
- [ ] 태그별 글 필터링
- [ ] 글 검색 기능 (제목, 내용, 작성자)
- [ ] 무한 스크롤 또는 페이지네이션
- [ ] 글 썸네일 이미지 표시

## 💬 소통 기능

### 댓글 시스템
- **독자로서** 글에 댓글을 남기고 싶다 **왜냐하면** 작성자와 소통하고 싶기 때문이다
- **독자로서** 다른 댓글에 답글을 달고 싶다 **왜냐하면** 다른 독자들과 토론하고 싶기 때문이다
- **블로거로서** 내 글의 댓글을 관리하고 싶다 **왜냐하면** 건전한 토론 환경을 만들고 싶기 때문이다

#### 수용 기준
- [ ] 로그인한 사용자만 댓글 작성 가능
- [ ] 댓글 작성 시 마크다운 지원
- [ ] 댓글에 답글 기능 (1단계 깊이)
- [ ] 댓글 수정/삭제 (작성자만)
- [ ] 댓글 신고 기능
- [ ] 글 작성자의 댓글 삭제 권한
- [ ] 댓글 작성 시간 표시

### 좋아요 및 북마크
- **독자로서** 좋은 글에 좋아요를 누르고 싶다 **왜냐하면** 작성자에게 감사를 표현하고 싶기 때문이다
- **독자로서** 나중에 읽을 글을 북마크하고 싶다 **왜냐하면** 시간이 있을 때 다시 읽고 싶기 때문이다
- **블로거로서** 내 글의 좋아요 수를 보고 싶다 **왜냐하면** 독자들의 반응을 알고 싶기 때문이다

#### 수용 기준
- [ ] 글 상세 페이지에 좋아요 버튼
- [ ] 좋아요 수 실시간 업데이트
- [ ] 중복 좋아요 방지 (사용자당 1회)
- [ ] 북마크 버튼 및 개인 북마크 목록
- [ ] 내가 좋아요한 글 목록 페이지
- [ ] 좋아요/북마크 상태 시각적 표시

## 👤 프로필 관리

### 개인 프로필
- **사용자로서** 내 프로필을 설정하고 싶다 **왜냐하면** 다른 사람들에게 나를 소개하고 싶기 때문이다
- **사용자로서** 프로필 사진을 업로드하고 싶다 **왜냐하면** 개성을 표현하고 싶기 때문이다
- **방문자로서** 작성자의 프로필을 보고 싶다 **왜냐하면** 작성자에 대해 더 알고 싶기 때문이다

#### 수용 기준
- [ ] 프로필 사진 업로드 (JPG, PNG, 최대 2MB)
- [ ] 닉네임, 자기소개 입력
- [ ] 소셜 링크 (GitHub, LinkedIn, Twitter 등)
- [ ] 전문 분야 태그 설정
- [ ] 프로필 공개/비공개 설정
- [ ] 다른 사용자 프로필 페이지 조회

### 개인 블로그 페이지
- **블로거로서** 내 블로그만의 페이지를 갖고 싶다 **왜냐하면** 개인 브랜딩을 하고 싶기 때문이다
- **방문자로서** 특정 작성자의 모든 글을 보고 싶다 **왜냐하면** 그 사람의 다른 글도 읽고 싶기 때문이다

#### 수용 기준
- [ ] 개인 블로그 URL (예: /blog/username)
- [ ] 작성자의 모든 공개 글 목록
- [ ] 프로필 정보 표시
- [ ] 글 카테고리별 분류
- [ ] 작성자 팔로우 기능 (향후 확장)

## 🔍 검색 및 발견

### 검색 기능
- **사용자로서** 키워드로 글을 검색하고 싶다 **왜냐하면** 원하는 정보를 빠르게 찾고 싶기 때문이다
- **사용자로서** 태그로 관련 글들을 모아보고 싶다 **왜냐하면** 특정 주제의 글들을 한번에 보고 싶기 때문이다

#### 수용 기준
- [ ] 전체 글 검색 (제목, 내용, 태그)
- [ ] 검색 결과 하이라이팅
- [ ] 검색 결과 정렬 (관련도, 최신순, 인기순)
- [ ] 태그 클릭 시 해당 태그 글 목록
- [ ] 인기 태그 목록 표시
- [ ] 검색어 자동완성 (향후 확장)

## 📱 반응형 및 접근성

### 모바일 최적화
- **모바일 사용자로서** 글을 편하게 읽고 싶다 **왜냐하면** 이동 중에도 콘텐츠를 소비하고 싶기 때문이다
- **모바일 사용자로서** 댓글을 쉽게 작성하고 싶다 **왜냐하면** 모바일에서도 소통하고 싶기 때문이다

#### 수용 기준
- [ ] 모바일 화면에 최적화된 레이아웃
- [ ] 터치 친화적인 버튼 크기
- [ ] 모바일 키보드 최적화
- [ ] 빠른 로딩 속도 (3초 이내)
- [ ] 오프라인 읽기 지원 (PWA, 향후 확장)

### 접근성
- **시각 장애인으로서** 스크린 리더로 글을 읽고 싶다 **왜냐하면** 콘텐츠에 접근하고 싶기 때문이다
- **키보드 사용자로서** 마우스 없이도 사이트를 이용하고 싶다 **왜냐하면** 편리하게 탐색하고 싶기 때문이다

#### 수용 기준
- [ ] 시맨틱 HTML 구조
- [ ] 적절한 alt 텍스트
- [ ] 키보드 네비게이션 지원
- [ ] 충분한 색상 대비
- [ ] 포커스 표시자 제공
