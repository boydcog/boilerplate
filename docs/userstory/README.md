# 유저스토리 (User Stories)

이 폴더에는 각 기능별 상세한 유저스토리를 저장합니다.

## 📁 폴더 구조

```
docs/userstory/
├── README.md           ← 이 파일
├── 로그인.md           ← 로그인 관련 유저스토리
├── 게시글관리.md        ← 게시글 CRUD 유저스토리
├── 사용자관리.md        ← 사용자 프로필 관리
└── 검색.md            ← 검색 기능 유저스토리
```

## 📝 유저스토리 작성 가이드

### 기본 형식
```
**사용자로서** [누가]
**~하고 싶다** [무엇을]
**왜냐하면** [왜]
```

### 예시
```markdown
# 로그인 기능

## 유저스토리
- **사용자로서** 이메일과 비밀번호로 로그인하고 싶다 **왜냐하면** 내 계정에 안전하게 접근하고 싶기 때문이다
- **사용자로서** 로그인 상태를 유지하고 싶다 **왜냐하면** 매번 로그인하는 것이 번거롭기 때문이다

## 수용 기준 (Acceptance Criteria)
- [ ] 올바른 이메일/비밀번호 입력 시 로그인 성공
- [ ] 잘못된 정보 입력 시 명확한 에러 메시지 표시
- [ ] 로그인 후 이전 페이지 또는 메인 페이지로 이동
- [ ] "로그인 유지" 체크박스 기능
- [ ] 비밀번호 찾기 링크 제공
```

## 🎯 작성 팁

### 좋은 유저스토리의 특징 (INVEST)
- **Independent**: 독립적이고 다른 스토리에 의존하지 않음
- **Negotiable**: 협상 가능하고 유연함
- **Valuable**: 사용자에게 가치를 제공
- **Estimable**: 개발 시간 추정 가능
- **Small**: 한 스프린트에 완료 가능한 크기
- **Testable**: 테스트 가능한 명확한 기준

### 수용 기준 작성 가이드
- 체크리스트 형태로 작성
- 테스트 가능한 구체적인 조건
- 긍정적인 케이스와 예외 상황 모두 포함
- 사용자 인터페이스 동작 명시
